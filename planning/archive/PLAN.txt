-----------------------------------------------------
3. CORE STACK
-----------------------------------------------------
Frontend     Next.js 15 (App Router) + TS + Tailwind v4
UI kit       shadcn/ui + custom Shadow tokens
Charts       Recharts (life wheel, trends)
Backend      Next.js API routes + Supabase
DB           Postgres + pgvector
Auth         Supabase Auth (email magic link)
LLM          Claude (Sonnet 4.6) primary, Haiku for fast classify
Embeddings   OpenAI text-embedding-3-small (or Voyage)
Hosting      Vercel
Analytics    PostHog (events) + own metrics tables

       Classification API route + Claude prompt

       Manual correction UI

       Daily questions modal w/ 5 random questions
Day 6  Daily report v0 (text only, no charts)
       Embeddings table + nightly job stub

-----------------------------------------------------
9. SUCCESS DEFINITION (MVP)
-----------------------------------------------------
  • I capture ≥3 entries/day for 2 weeks straight
  • Classification accuracy ≥80% subjective
  • Daily report opened ≥5 days/week
  • Weekly review feels actionable, not generic
  • Case study draft published to portfolio site

-----------------------------------------------------
10. RISKS
-----------------------------------------------------
  • AI generic output → fix via RAG context + low-confidence flags
  • Over-engineering → ship slim MVP schema first
  • Privacy → all data user-scoped via RLS, no analytics on raw text
  • Cost → Haiku for classification, Sonnet only for reports
  • Adoption (self) → daily questions reduce friction
  • Scope creep → freeze MVP scope after Phase 0 PRD

END.
