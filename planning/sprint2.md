PBI-04 — Реализовать страницу результата теста с AI-анализом. После завершения каждого теста пользователь должен получать не сухой результат вроде “ты интроверт на 72%”, а полноценную страницу анализа, где AI превращает сырые баллы и ответы в понятное описание личности, мотивации или текущего состояния. Результат должен включать главный вывод, карту показателей, поведенческие паттерны, объяснение значения результата для жизни, рекомендации по тому, как Shadow должен взаимодействовать с пользователем, memory candidates для сохранения в память и follow-up questions для дальнейшего раскрытия пользователя. Анализ не должен ставить диагнозы, не должен имитировать медицинское заключение и должен явно содержать safety-note о том, что Labs — это инструмент самонаблюдения и персонализации AI, а не медицинская диагностика.
PBI-05 — Добавить AI Summary в профиль пользователя. В разделе Profile реализовать блок AI Summary, который показывает, как Shadow на текущий момент понимает пользователя. AI Summary должен собираться из Labs, Daily Check-in, ответов на AI-вопросы, важных инсайтов, целей, привычек, памяти и выявленных паттернов. В summary должны отражаться core personality, motivation drivers, communication style, current state, planning style, risk zones и Shadow interaction rules. Система должна разделять стабильную информацию о личности и временное текущее состояние, чтобы плохой день или временная усталость не перезаписывали устойчивый психологический профиль пользователя. AI Summary должен иметь дату последнего обновления и обновляться после значимых событий.
PBI-06 — Реализовать AI Brain как центральный слой персонализации. Создать AI Brain — внутренний слой Shadow, который перед каждым AI-ответом собирает context pack и решает, что сейчас известно о пользователе, что важно в текущий момент, какие сигналы свежие, какие воспоминания релевантны, какие цели активны, какие паттерны повторяются, какие инициативы актуальны и какие вопросы стоит задать дальше. AI Brain должен использовать данные из Profile AI Summary, Labs results, Daily Check-ins, Inbox, Goals, Tasks, Habits, Insights, Analytics, Memory Items, Memory Graph, active initiatives и conversation context. Цель AI Brain — сделать ответы Shadow персонализированными, контекстными и полезными, а не универсальными советами.
PBI-07 — Реализовать разные типы памяти пользователя. Разделить память Shadow на несколько типов, чтобы AI мог точнее понимать природу информации. Система должна поддерживать Profile Memory для устойчивых данных о пользователе, Episodic Memory для событий и ситуаций, Behavioral Memory для повторяющихся паттернов, Goal Memory для целей и проектов, Current State Memory для временного состояния, Preference Memory для предпочтений, Insight Memory для важных мыслей и Relationship Memory для информации о людях и отношениях. Каждая запись памяти должна иметь тип, источник, важность, стабильность, confidence, tags, timestamps и связь с исходным объектом, из которого она была создана.
PBI-08 — Реализовать RAG Memory для семантического поиска по памяти. Создать систему memory_items, которая сохраняет не все сырые ответы пользователя, а только значимые обобщённые инсайты, пригодные для будущей персонализации. Каждая memory item должна быть доступна для vector retrieval через embeddings и использоваться AI Brain при построении context pack. При поиске релевантной памяти система должна учитывать семантическую близость, тип памяти, свежесть, стабильность, confidence, importance и текущий контекст запроса. Система не должна засорять память отдельными ответами вроде “пользователь выбрал +4 на вопрос 12”; вместо этого она должна сохранять смысловые выводы вроде устойчивых мотивационных драйверов, предпочтительного стиля общения, рисков, паттернов и текущих ограничений пользователя.
PBI-09 — Реализовать Memory Graph как структуру связей между данными пользователя. Помимо плоской RAG-памяти, создать memory graph, где важные элементы жизни пользователя сохраняются как nodes, а связи между ними — как edges. Memory Graph должен поддерживать node types: User Profile, Value, Goal, Project, Habit, Emotion, Event, Pattern, Risk, Preference, Person, Place, Decision, Insight и Current State. Edge types должны включать supports, blocks, triggers, repeats_in, belongs_to, contradicts, strengthens, weakens, causes и related_to. Граф должен позволять Shadow понимать не только отдельные факты, но и связи между ценностями, целями, эмоциями, привычками, рисками и повторяющимися паттернами. MVP может подготовить данные без визуализации, но структура должна быть готова для будущего отображения визуального графа памяти пользователю.
PBI-10 — Реализовать Knowledge Gaps. Добавить в AI Brain сущность Knowledge Gaps — пробелы в понимании пользователя, которые мешают Shadow взаимодействовать с ним точнее. Система должна определять, чего Shadow ещё не знает о пользователе, на основе целей, поведения, пропусков, неоднозначных сигналов, повторяющихся тем и неполных данных. Knowledge Gap может появляться, когда Shadow знает, что пользователь хочет быть продуктивнее, но не знает, что его блокирует; знает, что пользователь хочет развивать Shadow, но не знает, что его мотивирует; знает, что пользователь пропускает приложение, но не знает почему; знает, что у пользователя много идей, но не знает, как он выбирает приоритеты. Каждый Knowledge Gap должен иметь reason, source, status, priority и возможность породить один короткий полезный AI-вопрос.
PBI-11 — Реализовать AI Questions как способ мягко раскрывать пользователя. Создать систему AI Questions, где Shadow иногда задаёт пользователю короткие, точные и полезные вопросы, основанные на Knowledge Gaps, паттернах, текущем состоянии или целях. Вопросы должны помогать пользователю делиться важной информацией о себе без необходимости самому понимать, какие факты важны. Система должна поддерживать типы вопросов: identity, motivation, friction, pattern, context, reflection, goal, habit и emotional state. Вопрос задаётся только один за раз, пользователь может ответить, пропустить, отложить или закрыть его, а ответ может стать memory candidate, обновить AI Summary, создать memory item, node в memory graph или новую инициативу.
PBI-12 — Реализовать Daily Check-in как главный ежедневный activation loop. Добавить глобальную кнопку Daily Check-in, доступную из Dashboard, sidebar, floating assistant и мобильной навигации, чтобы у пользователя была понятная причина возвращаться в Shadow каждый день. Daily Check-in должен быть не скучной формой, а красивым ежедневным ритуалом синхронизации с Shadow, который занимает 2–5 минут и помогает пользователю выгрузить состояние, мысли, фокус, привычки, цели и инсайты. Flow должен включать Current State, Inbox Dump, Today Focus, Habits/Souls, Goals, Insight, Optional AI Question и Today Initiative. После завершения пользователь должен видеть экран Daily Sync Complete с указанием, что current state обновлён, memory signals captured, initiative generated, goal context refreshed и Souls streak protected.
PBI-13 — Реализовать Current State внутри Daily Check-in. В Daily Check-in добавить быстрый блок оценки текущего состояния, где пользователь оценивает energy, mood, mental noise, body state и focus. Energy, mental noise, body state и focus использовать по шкале 0–5, mood — по шкале -5 до +5. Эти данные должны сохраняться в daily_checkins и использоваться AI Brain для адаптации ответов, инициатив, планов и рекомендаций. Текущее состояние должно храниться отдельно от стабильного профиля личности, чтобы Shadow мог понимать, что сегодня пользователь может быть уставшим, перегруженным или расфокусированным, не меняя при этом его базовый психологический портрет.
PBI-14 — Реализовать Inbox Dump внутри Daily Check-in. В Daily Check-in добавить свободное текстовое поле для выгрузки того, что сейчас крутится в голове пользователя: мысли, задачи, тревоги, идеи, решения, хаос, инсайты или события. Ввод должен сохраняться как часть daily_checkin и/или как inbox entry, после чего AI может классифицировать его по типам, связать с целями, привычками, памятью, инициативами и текущим состоянием. Этот блок нужен, чтобы снизить барьер входа: пользователь не обязан заранее структурировать свою мысль, он просто выгружает её, а Shadow потом помогает превращать хаос в структуру.
PBI-15 — Реализовать Today Focus внутри Daily Check-in. В Daily Check-in добавить выбор главного фокуса дня, чтобы Shadow понимал, какую стратегию взаимодействия выбрать. Пользователь должен иметь возможность выбрать один основной фокус из вариантов вроде сделать конкретную задачу, восстановиться, разобраться в себе, продвинуть проект, закрыть хвост, просто пережить день или указать свой вариант. Выбранный фокус должен использоваться AI Brain для генерации Today Initiative, персонализации рекомендаций, выбора тона и определения того, стоит ли давать пользователю амбициозный план, мягкое восстановление, рефлексию или минимальное действие.
PBI-16 — Интегрировать Habits и Souls в Daily Check-in. В Daily Check-in добавить шаг, где пользователь выбирает минимальный ритуал или привычку, которую готов защитить сегодня. Этот шаг должен быть связан с системой привычек и Souls-механикой, где Souls накапливаются за выполнение и могут обнуляться после длительного отсутствия действий. Shadow должен использовать check-in, чтобы напоминать о минимальном действии, защищать streak, снижать порог выполнения и превращать привычки в живой игровой ритуал прогресса, а не в сухую таблицу.
PBI-17 — Связать Daily Check-in с Goals. В Daily Check-in добавить шаг выбора активной цели, к которой относится сегодняшний фокус, мысль, действие или состояние. Shadow должен показывать активные цели пользователя и позволять связать check-in с одной из них или пропустить этот шаг. Эта связь должна сохраняться в linked_goal_id и использоваться AI Brain для понимания, куда направлена энергия пользователя, какие цели получают внимание, какие цели простаивают и какие инициативы стоит предложить.
PBI-18 — Реализовать Insight Capture внутри Daily Check-in. Добавить в Daily Check-in короткий блок, где пользователь может сохранить одну мысль, которую стоит запомнить. Этот insight должен сохраняться как отдельный сигнал и после AI-обработки может стать memory item, node в memory graph, обновлением AI Summary или источником будущей инициативы. Цель блока — помочь пользователю постепенно формировать личную базу смыслов, решений, наблюдений и внутренних выводов без необходимости открывать отдельный раздел памяти.
PBI-19 — Реализовать Shadow Initiatives. Создать систему Shadow Initiatives — проактивных наблюдений, вопросов, подсказок и действий, которые Shadow генерирует на основе сигналов пользователя. Инициативы должны быть не случайными пушами, а обоснованными предложениями, где есть title, reason, pattern_duration, suggested_action, linked_area, source_signals, priority, status, expiration date и timestamps. Поддержать типы инициатив: Observation Initiative, Question Initiative, Pattern Initiative, Recovery Initiative, Goal Initiative, Habit Initiative, Memory Gap Initiative, Productive Nudge и Reflection Prompt. Пользователь должен иметь возможность принять, выполнить, отложить, закрыть или проигнорировать инициативу, а система должна учитывать эти действия в будущем.
PBI-20 — Реализовать pattern duration для инициатив и аналитики. Добавить в Shadow Initiatives и AI Brain логику длительности паттернов, чтобы система могла различать краткосрочный сигнал и глубокую повторяющуюся тенденцию. Поддержать уровни Micro-pattern на 2 дня, Short pattern на 3–7 дней, Medium pattern на 2–4 недели, Long pattern на 1–3 месяца и Core pattern как долгосрочно повторяющийся паттерн. При генерации инициативы Shadow должен уметь объяснить, почему он считает сигнал паттерном, сколько он длится, какие источники это подтверждают и почему сейчас стоит обратить на него внимание.
PBI-21 — Реализовать Today Initiative после Daily Check-in. После завершения Daily Check-in Shadow должен генерировать одну главную Today Initiative, основанную на current state, today focus, active goals, recent check-ins, habits, memory, knowledge gaps и повторяющихся сигналах. Today Initiative должна быть короткой, конкретной, обоснованной и практически применимой. Она должна помогать пользователю понять, что делать дальше, не создавая перегруз. Если пользователь в ресурсе, инициатива может предлагать продуктивное действие; если перегружен — восстановление или минимальный шаг; если наблюдается важный паттерн — рефлексию или уточняющий вопрос.
PBI-22 — Реализовать Floating Shadow Assistant. Добавить floating assistant справа снизу в интерфейсе Shadow как Shady / Shadow Orb — живого, спокойного, ненавязчивого AI-компаньона, а не обычный generic chat bubble. Ассистент должен уметь мягко появляться при входе в приложение, после бездействия, после завершения теста, после Daily Check-in, при открытии релевантной вкладки, при наличии важной инициативы или knowledge gap. Он может показывать короткое наблюдение, задать один вопрос, предложить посмотреть инициативу, спросить быструю оценку энергии или помочь превратить инсайт в действие.
PBI-23 — Добавить anti-annoyance rules для Floating Assistant и инициатив. Чтобы Shadow не стал раздражающим ассистентом, реализовать ограничения проактивности: максимум одно активное всплытие за сессию, максимум три proactive prompts в день, всегда доступные Not now, Snooze и Dismiss, возможность отключить proactive mode, правило “один вопрос за раз” и запрет на агрессивные или слишком частые вмешательства. Shadow должен ощущаться как тихий аналитик и спутник, а не как навязчивый бот, поэтому каждая проактивная реплика должна иметь reason, контекст и полезность.
PBI-24 — Реализовать связку Labs → AI Analysis → Memory → Profile AI Summary. После прохождения любого теста в Labs система должна сохранять сырые ответы, рассчитывать scores, генерировать AI-анализ, выделять behavioral patterns, формировать memory candidates, сохранять значимые инсайты в RAG Memory, создавать или обновлять nodes/edges в Memory Graph и обновлять Profile AI Summary. Вся цепочка должна работать как единый pipeline, чтобы Labs не были изолированной вкладкой, а становились источником долгосрочной персонализации Shadow.
PBI-25 — Реализовать связку Daily Check-in → AI Brain → Initiatives → Memory. После Daily Check-in система должна сохранять состояние, inbox dump, фокус дня, связь с целью, привычкой и insight, затем передавать эти сигналы в AI Brain. AI Brain должен определить важные сигналы, обновить current state, создать memory candidates, проверить knowledge gaps, обнаружить паттерны, обновить AI Summary при необходимости и сгенерировать Today Initiative. Daily Check-in должен стать ежедневным источником данных, который делает Shadow всё точнее и полезнее.
PBI-26 — Реализовать связку Knowledge Gaps → AI Questions → Memory Update. Когда AI Brain обнаруживает, что для лучшего понимания пользователя не хватает важной информации, система должна создать Knowledge Gap, сгенерировать один короткий вопрос и показать его через Daily Check-in, Floating Assistant или подходящую вкладку. Ответ пользователя должен сохраняться, классифицироваться, превращаться в memory candidate и при достаточной значимости обновлять RAG Memory, Memory Graph, AI Summary или будущие инициативы. Цель — сделать так, чтобы Shadow сам помогал пользователю раскрывать свою историю, мотивацию, ограничения и паттерны, а не ждал идеального ручного ввода.
PBI-27 — Подготовить базу данных для Shadow Intelligence Loop v1. Создать или адаптировать таблицы для Labs, AI Brain, памяти, графа, check-in, вопросов и инициатив. Нужны tables: labs_tests, labs_questions, labs_answer_options, labs_sessions, labs_answers, labs_results, profile_ai_summary, memory_items, memory_graph_nodes, memory_graph_edges, daily_checkins, ai_questions, ai_question_answers и shadow_initiatives. База должна хранить тесты, вопросы, ответы, результаты, AI-интерпретации, профильные summary, memory items, graph nodes, graph edges, daily signals, AI questions, answers and initiatives. Структура должна быть расширяемой, чтобы позже добавить новые тесты, визуализацию графа, более сложную аналитику, уведомления и персонализированные сценарии.
PBI-28 — Добавить safety и trust layer для Labs и AI-анализа. Во всех местах, где Shadow анализирует личность, состояние или психологические паттерны, добавить понятный safety layer: Shadow Labs не является медицинской диагностикой, терапией или клиническим инструментом, а используется для самонаблюдения, структурирования информации и улучшения AI-персонализации. AI-выводы должны быть сформулированы осторожно, без диагнозов, категоричных утверждений и манипулятивного языка. Пользователь должен понимать, какие данные сохраняются, зачем они используются, и иметь контроль над памятью, инициативами и проактивным поведением ассистента.
PBI-29 — Создать closed personalization loop. Объединить Labs, Daily Check-in, AI Brain, Knowledge Gaps, AI Questions, RAG Memory, Memory Graph, Profile AI Summary, Shadow Initiatives и Floating Assistant в единый цикл персонализации. Пользователь проходит тесты и check-in, Shadow собирает структурные и ежедневные данные, AI Brain определяет смысл, память обновляется, граф связей растёт, AI Summary становится точнее, инициативы становятся полезнее, а пользователь получает ощущение, что Shadow понимает его всё лучше с каждым использованием. Главный продуктовый эффект: “Every time I use Shadow, it understands me better.”
PBI-30 — Усилить ежедневную мотивацию возвращаться в Shadow. Решить проблему, что пользователю сейчас нужно заставлять себя заходить в приложение, через создание ежедневного ритуала, проактивного ассистента и видимого накопления пользы. Shadow должен давать маленькую причину вернуться каждый день: пройти Daily Check-in, защитить Souls streak, ответить на один полезный вопрос, увидеть Today Initiative, обновить состояние, сохранить инсайт или улучшить AI Summary. Приложение должно создавать ощущение живой системы, которая развивается вместе с пользователем и становится полезнее от каждого входа, а не просто хранит данные.
